# YouTube 英文字幕取词设计

## 目标与边界

在普通 YouTube `/watch` 录播视频中，为当前可见的人工或自动英文字幕提供稳定、准确的取词
入口。首版覆盖普通、剧院和全屏模式，不支持直播、Shorts、OCR、整段字幕下载、跨字幕拼句
或字幕历史。

功能只读取 YouTube 已渲染的当前字幕 DOM，不调用 YouTube Data API，不申请 OAuth，也不新增
Chrome 权限。YouTube DOM 适配失败时功能静默不可用，普通网页划词必须继续工作。

## 交互

Huayi 在 YouTube CC 按钮旁显示“译”按钮。没有当前英文字幕、处于广告或直播状态时按钮禁用。
用户点击后：

1. 若视频正在播放，Huayi 暂停视频并记录需要恢复；原本暂停的视频不记录恢复。
2. 当前可见字幕被冻结为播放器底部居中的 Shadow DOM 卡片。
3. 单击准确选择单词；按住鼠标跨单词拖动选择连续短语；“整条字幕”选择当前可见字幕。
4. “解释｜翻译”工具条、加载状态和结果卡锚定触发操作时的鼠标位置：点词使用单击位置，
   拖选使用松开位置，“整条字幕”使用该按钮的点击位置；浮层水平中心与该位置对齐，优先
   向上展开，空间不足时换边并钳制在视口内。键盘激活时回退到对应按钮中心。结果卡仍可
   拖动。
5. 冻结卡片保留字幕和选中高亮，允许在同一次暂停中连续查询其他单词。

再次点击处于激活状态的“译”、“继续播放”、关闭和 Escape 使用同一关闭路径结束取词。
只有 Huayi 主动暂停的视频会恢复。用户主动播放、拖动进度、视频结束、播放器被替换或离开
`/watch` 时，界面和活动请求立即清理且不强制播放。

## 文本与请求

当前可见 `.ytp-caption-segment` 必须有可见布局框；按视觉顺序合并后规范空白、拒绝汉字和超过
2,000 字符的内容。分词优先使用 `Intl.Segmenter("en", { granularity: "word" })`，并合并英文
单词内部的撇号和连字符；回退正则保持相同边界。

合成请求继续使用 wire v5：

- `selection` 是选中的单词、连续范围或整条当前字幕；
- `context` 是冻结字幕；
- `selectionKind` 使用既有分类器；
- 单词和短语的 `sentenceContext` 是冻结字幕；
- 仅单词的 `wordbookContext` 是冻结字幕。

YouTube 页面标题属于普通网页选区，不进入冻结字幕流程。标准 `h1` 至 `h6` 标题作为有界语义
文本块参与既有句子提取，因此标题内单词可携带完整英文标题作为 `sentenceContext` 和
`wordbookContext`，继续显示欧路生词本动作；该规则同样适用于其他网页的标准标题，不采集
YouTube 页面元数据或更大的自定义容器。

进入取词只发送既有无页面数据 `warmup`。只有点击解释或翻译才发送上述文本；单词结果继续
复用既有欧路查词和显式加入流程。

## 实现与安全

YouTube 选择器、SPA 事件、字幕观察、播放器状态和 UI 全部隔离在 Extension 的 YouTube 模块。
普通网站不创建该观察器。Control、冻结卡和结果使用原生 DOM、开放 Shadow DOM 与
`textContent`，不使用远程代码或 `innerHTML`。

结果浮层通过可选展示策略解析实时锚点和挂载节点：普通模式挂载文档根节点；全屏模式挂载
`document.fullscreenElement`。默认展示策略保持普通网页现有行为。

不记录 URL、标题、字幕、查询或结果，不缓存之前/之后的字幕，不把字幕发送给 warmup。Native
Host、Provider、凭据、Manifest 权限、版本 `0.10.0` 和 `schemaVersion: 5` 均不变。

## 验收

默认离线测试覆盖可见字幕读取、动态节点、英文过滤、分词、精确上下文、智能暂停恢复、连续
取词、鼠标位置锚定、“译”按钮切换关闭、优先向上定位、全屏重挂载和普通网页回归。
Playwright 使用本地 YouTube DOM fixture 验证真实 Content Script、Service Worker、请求
协调器和 fake Native Host 全链路；真实 YouTube/Chrome 验证需单独授权。
