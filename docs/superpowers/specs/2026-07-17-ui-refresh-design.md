# v0.9.0 浮层视觉改造设计

## 目标

把划译从灰色、多层小卡片的工具面板改造成阅读场景中的轻量词典卡。视觉改造不能改变 wire
v5、Provider、模型请求、欧路数据、Chrome 权限或结果语义。

设计参考以下成熟模式：DeepL 使用紧贴阅读上下文、可拖动的小窗口；Apple 建议浮层只承载少量
相关任务、避免过大并平滑处理尺寸变化；Carbon 要求单一浮层、完整键盘操作并避免嵌套浮层。

## 视觉层级

- 工具条保持“解释/翻译”两个动作，使用白色表面、克制阴影和带字形的小型强调图标。
- 结果卡使用单一白色表面，宽度上限 420px、圆角 16px，不在结果项外重复套卡片。
- 顶部模式标题降为次要标签；单词源词使用 21px 词典头部，音标紧随其后。
- 语境义或语境解析是唯一使用柔和蓝色底和左侧强调线的内容区。
- 常见释义使用词性胶囊与释义行；短语、词形和用法使用左右结构化行；易混词和同义词使用
  双列对照，320px 下退化为单列。
- 欧路动作固定在右上角，使用书签加号图标和“生词/添加中/已加入”短标签；完整含义通过
  `aria-label` 暴露。

## 流式与状态

- 继续按 `data-huayi-section` 键控 patch，源词、音标、板块和数组项不得因增量更新整卡替换。
- 加载和单词流式阶段保留 208px 最小正文高度，降低前几个板块到达时的纵向跳动。
- 新节点只执行一次 120ms 淡入和 3px 位移；`prefers-reduced-motion` 下禁用动画与过渡。
- 最终空板块继续隐藏；错误预览、生词错误、重试、焦点、滚动和拖动语义保持不变。

## 无障碍与窄屏

- Escape、键盘拖动、可见焦点和视口约束保持现有行为。
- 320px 下卡片宽度为视口减 16px；短语与辨析改为单列，头部生词、拖动和关闭控件不得重叠。
- 所有网页与模型文本仍只使用 `textContent`；SVG 仅包含固定本地图标路径。

## 非目标

- 不增加暗色模式、设置页、朗读、复制、折叠区、侧边栏或第二种主题。
- 不修改单词、短语、句子或段落的模型契约和字段顺序。
- 不执行真实模型 smoke；本版只需离线测试、浏览器视觉回归、构建和本机同步替换。
